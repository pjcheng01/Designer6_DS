## 目前進行中的問題

### `system.ini` 中文 key 無法讀取

`getfile_val` 函式在 DraftSight 2025（UTF-8 環境）下無法比對 `system.ini` 的中文 key，回傳 `nil`。

**測試確認：**
```lisp
(getfile_val (strcat POWDESIGN_path "SYSTEM.ini") "零件定義資料")
;; 結果：nil
```

`system.ini` 已是 UTF-8，內容正常，但 `getfile_val` 的 `strcase` 比對邏輯有問題。

**受影響的功能：**
- `c:automakepart`（自動建立資訊點）
- `SYSTEM.lsp` 的 `getval` 函式（圖層、線型定義）
- `MANAPART.lsp` 的多個讀取 `system.ini` 的地方

**根本解法：**
把 `system.ini` 的中文 key 全部改為英文 key，同時更新所有讀取這些 key 的 lsp 程式。
這跟之前處理 `config.doc` 的方式相同。

## DraftSight 環境設定

- **版本**：DraftSight 2025 Enterprise
- **Support Files Search Path**：
  - `C:\DESIGNER6_DS`
  - `C:\POWPARTS_DS`
- **選單載入**：`CUSTOMIZE` → 載入 `C:\DESIGNER6_DS\Powsoft_Light.xml`
- **自動載入**：`startup.lsp` 由 DraftSight 的 `start.lsp` 自動執行

## 啟動流程

```
DraftSight 啟動
  → start.lsp（DraftSight 內建）
    → startup.lsp（C:\DESIGNER6_DS）
      → (load "command")   → COMMAND.lsp
      → (load "quickey")   → QUICKEY.lsp
      → (c:autoload)
          → (load "designer") → designer.lsp
              → (load "PUB-LISP")
              → (load "config") → config.lsp
                  → 讀取 config.doc（英文 key）
                  → 設定 POWDESIGN_path、POWPARTS_path 等
          → (loaddesigner) → SYSTEM.lsp
              → get_layerdef()   ← 預設值
              → get_ltypedef()   ← 預設值
              → get_bomdef()     ← 預設值
              → get_bomlistdef() ← 預設值
```
